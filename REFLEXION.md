Reflexión:

1. El CI evita principalmente errores técnicos y de integración, como:
- Errores en pruebas automatizadas
- Errores de lógica introducidos por cambios recientes
- Respuestas incorrectas de una API
- Código que funciona localmente pero falla al integrarse con el repositorio.

2. El CI no evitas errores relacionados con requisitos mal definidos o incompletos, problemas de experiencia de usuario,
errores lógicos que no estén cubiertos por pruebas y decisiones de diseño incorrectas.
El CI puede verificar la calidad técnica pero no las decisiones humanas ni la validción funcional completa

 3. Si un equipo ignora el CI se pueden integrar cambios defectuosos al código base, también aumenta el riesgo de errores en producción,
se pierde confianza en el sistema y en el equipo y el mantenimiento se vuelve más costoso y lento.
Ignorar el CI debilita la calidad del software y va en contra de las buenas prácticas de desarrollo web
